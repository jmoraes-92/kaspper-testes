1. Tabela usuarios

Representa os usuários do sistema, que podem ter diferentes perfis.

  • Atributos:
  • id_usuario (BIGINT): Chave primária única para cada usuário.
  • data_criacao (DATETIME): Data e hora de criação do registro.
  • email (VARCHAR(150)): E-mail do usuário.
  • nome (VARCHAR(100)): Nome do usuário.
  • perfil (ENUM): Define o perfil do usuário (e.g., Administrador, Operador).
  • senha (VARCHAR(255)): Senha para autenticação.
  • Relacionamentos:
  • Está relacionada com a tabela clientes pelo atributo id_usuario (um usuário pode estar associado a vários clientes).
  • Relaciona-se com a tabela notificacoes (um usuário pode receber várias notificações).

2. Tabela clientes

Representa os clientes associados a um usuário.

  • Atributos:
  • id_cliente (BIGINT): Chave primária única para cada cliente.
  • data_cadastro (DATETIME): Data e hora de cadastro do cliente.
  • email (VARCHAR(255)): E-mail do cliente.
  • empresa (VARCHAR(255)): Nome da empresa do cliente.
  • nome (VARCHAR(255)): Nome do contato principal.
  • telefone (VARCHAR(255)): Telefone do cliente.
  • id_usuario (BIGINT): Chave estrangeira que referencia a tabela usuarios.
  • Relacionamentos:
  • Está relacionada com a tabela usuarios (um cliente é gerenciado por um usuário).
  • Relaciona-se com a tabela demandas (um cliente pode ter várias demandas associadas).
  • Relaciona-se com a tabela notificacoes (notificações podem ser enviadas para clientes).

3. Tabela demandas

Registra solicitações ou tarefas realizadas pelos clientes.

  • Atributos:
  • id_demanda (BIGINT): Chave primária única para cada demanda.
  • data_criacao (DATETIME): Data e hora de criação da demanda.
  • descricao (TEXT): Descrição detalhada da demanda.
  • prioridade (ENUM): Define a prioridade da demanda (e.g., Alta, Média, Baixa).
  • status (ENUM): Define o status da demanda (e.g., Aberta, Em andamento, Concluída).
  • id_cliente (BIGINT): Chave estrangeira que referencia a tabela clientes.
  • Relacionamentos:
  • Relacionada com a tabela clientes (uma demanda pertence a um cliente).
  • Relaciona-se com a tabela tarefas (uma demanda pode ter várias tarefas associadas).
  • Relaciona-se com a tabela orcamentos (um orçamento pode ser gerado para uma demanda).

4. Tabela tarefas

Armazena as tarefas detalhadas relacionadas a uma demanda.

  • Atributos:
  • id_tarefa (BIGINT): Chave primária única para cada tarefa.
  • descricao (VARCHAR(255)): Descrição da tarefa.
  • prazo (DATE): Prazo para conclusão da tarefa.
  • responsavel (VARCHAR(255)): Nome do responsável pela tarefa.
  • status (TINYINT): Status da tarefa (e.g., 0 para pendente, 1 para concluída).
  • id_demanda (BIGINT): Chave estrangeira que referencia a tabela demandas.
  • Relacionamentos:
  • Relacionada com a tabela demandas (uma tarefa pertence a uma demanda).

5. Tabela orcamentos

Armazena informações sobre orçamentos relacionados às demandas.

  • Atributos:
  • id_orcamento (BIGINT): Chave primária única para cada orçamento.
  • observacoes (TEXT): Observações adicionais sobre o orçamento.
  • data_geracao (DATETIME): Data e hora de geração do orçamento.
  • status (ENUM): Define o status do orçamento (e.g., Pendente, Aprovado, Rejeitado).
  • valor (DECIMAL(38,2)): Valor do orçamento.
  • prazo_estimado (INT): Prazo estimado em dias para execução.
  • id_demanda (BIGINT): Chave estrangeira que referencia a tabela demandas.
  • Relacionamentos:
  • Relacionada com a tabela demandas (um orçamento está vinculado a uma demanda).
  • Relaciona-se com a tabela notificacoes (notificações podem ser enviadas sobre o status do orçamento).

6. Tabela notificacoes

Armazena notificações enviadas a usuários e clientes.

  • Atributos:
  • id_notificacao (BIGINT): Chave primária única para cada notificação.
  • data_envio (DATETIME): Data e hora de envio da notificação.
  • mensagem (TEXT): Mensagem enviada na notificação.
  • visualizada (BIT(1)): Define se a notificação foi visualizada (0 para não, 1 para sim).
  • id_usuario (BIGINT): Chave estrangeira que referencia a tabela usuarios.
  • id_cliente (BIGINT): Chave estrangeira que referencia a tabela clientes.
  • id_orcamento (BIGINT): Chave estrangeira que referencia a tabela orcamentos.
  • Relacionamentos:
  • Relacionada com as tabelas usuarios, clientes e orcamentos.

Resumo dos Relacionamentos

  1. Usuários - Clientes: Um usuário pode gerenciar vários clientes (1:N).
  2. Clientes - Demandas: Um cliente pode registrar várias demandas (1:N).
  3. Demandas - Tarefas: Uma demanda pode ter várias tarefas associadas (1:N).
  4. Demandas - Orçamentos: Cada demanda pode gerar um orçamento (1:1 ou 1:N, dependendo da regra de negócio).
  5. Notificações:
  • Relacionada a usuários (1:N).
  • Relacionada a clientes (1:N).
  • Relacionada a orçamentos (1:N).
